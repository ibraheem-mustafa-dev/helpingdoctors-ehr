# Conversation Extracts

This folder stores extracted decisions and context from previous Claude conversations.

---

## Purpose

When important decisions or context emerge from a conversation, extract them here so they're preserved for future sessions.

---

## File Format

```markdown
# Extract: [Topic]
*Source: [Conversation name/date]*
*Extracted: [Date]*

## Context
[Why this decision was made]

## Decision
[What was decided]

## Implementation
[How it was/should be implemented]

## References
- [Related files]
- [Related specifications]
```

---

## Example Extracts

Potential extracts to create:
- `extract-widget-count-53.md` - The 53 vs 16 widget clarification
- `extract-um-roles-only.md` - Why UM roles, not WordPress roles
- `extract-offline-first.md` - Why offline capability is critical

---

## When to Create Extracts

Create an extract when:
1. A significant decision is made that affects architecture
2. A common misunderstanding is corrected
3. Context is established that should persist across sessions
4. A precedent is set for how to handle similar situations

---

*This folder should be populated from conversation analysis as patterns emerge.*
